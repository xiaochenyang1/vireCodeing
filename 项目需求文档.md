# 货运接单平台 - 项目需求文档

## 项目概述

开发一个货运接单平台的移动应用（Android），连接货主和车主，实现货运订单的发布、接单、运输和结算全流程。

---

## 用户角色

### 1. 货主端（发货用户）
需要运输货物的个人或企业用户

### 2. 车主端（司机）
拥有货车并提供运输服务的司机

---

## 一、货主端功能模块

### 1.1 用户注册与登录
- [ ] 手机号注册/登录
- [ ] 验证码登录
- [ ] 实名认证（姓名、身份证）
- [ ] 企业认证（可选：营业执照、税号）
- [ ] 个人信息管理

### 1.2 发布订单
- [ ] **货物信息**
  - 货物类型（建材、食品、家电、化工等）
  - 货物重量
  - 货物体积
  - 货物数量
  - 是否需要装卸工
  - 货物照片上传

- [ ] **地址信息**
  - 装货地址（地图选点 + 详细地址）
  - 卸货地址（地图选点 + 详细地址）
  - 装货联系人和电话
  - 卸货联系人和电话

- [ ] **车辆要求**
  - 车型选择（小货车、中型货车、大货车、厢式货车等）
  - 车长要求
  - 是否需要尾板

- [ ] **时间要求**
  - 期望装货时间
  - 期望送达时间
  - 立即用车 / 预约用车

- [ ] **价格设置**
  - 系统智能估价
  - 货主自定义报价
  - 一口价 / 议价模式

### 1.3 订单管理
- [ ] **订单状态**
  - 待接单：查看司机报价，选择司机
  - 待装货：查看司机信息和位置
  - 运输中：实时追踪货物位置
  - 待确认：确认送达
  - 已完成：评价司机
  - 已取消：查看取消原因

- [ ] **订单操作**
  - 查看司机资料和评分
  - 联系司机（电话、聊天）
  - 取消订单（根据状态收取不同违约金）
  - 修改订单（装货前可修改）
  - 追加小费

### 1.4 支付与结算
- [ ] 多种支付方式
  - 微信支付
  - 支付宝
  - 银行卡
  - 企业对公账户
- [ ] 在线支付 / 货到付款
- [ ] 订单发票申请
- [ ] 消费记录查询

### 1.5 其他功能
- [ ] **常用地址管理**
  - 保存常用装货/卸货地址
  - 快速选择地址

- [ ] **历史订单**
  - 订单筛选和搜索
  - 重新下单

- [ ] **评价系统**
  - 对司机进行评分和评价
  - 查看自己的评价记录

- [ ] **客服与帮助**
  - 在线客服
  - 常见问题
  - 投诉与建议

- [ ] **消息通知**
  - 订单状态变更通知
  - 司机接单通知
  - 支付提醒

---

## 二、车主端（司机端）功能模块

### 2.1 司机注册与认证
- [ ] 手机号注册/登录
- [ ] **实名认证**
  - 姓名、身份证
  - 人脸识别验证

- [ ] **车辆认证**
  - 车牌号
  - 行驶证照片
  - 车辆类型（小货车、中型货车、大货车等）
  - 车长、载重
  - 是否有尾板

- [ ] **资质认证**
  - 驾驶证照片
  - 道路运输从业资格证
  - 营运证

- [ ] **银行卡绑定**（用于收款）

### 2.2 接单与抢单
- [ ] **订单大厅**
  - 查看附近的货运订单
  - 按距离、价格、时间筛选
  - 订单详情查看（货物、地址、价格）

- [ ] **接单模式**
  - 抢单模式：先到先得
  - 竞价模式：司机报价，货主选择

- [ ] **订单提醒**
  - 新订单推送通知
  - 附近订单提醒

### 2.3 订单执行
- [ ] **订单状态**
  - 待装货：查看装货地址和联系人
  - 导航到装货点（调用地图导航）
  - 装货完成确认（拍照上传）
  - 运输中：导航到卸货点
  - 卸货完成确认（拍照上传）
  - 等待货主确认
  - 已完成：收到款项

- [ ] **订单操作**
  - 联系货主（电话、聊天）
  - 上报异常（货物损坏、地址错误等）
  - 取消订单（特殊情况，需说明原因）

### 2.4 收入与提现
- [ ] **收入明细**
  - 今日收入
  - 本周收入
  - 本月收入
  - 历史收入

- [ ] **提现功能**
  - 余额提现到银行卡
  - 提现记录查询

- [ ] **收入统计**
  - 订单数量统计
  - 行驶里程统计

### 2.5 其他功能
- [ ] **接单设置**
  - 接单开关（在线/离线）
  - 接单范围设置（距离）
  - 接单类型设置（车型匹配）

- [ ] **评价系统**
  - 对货主进行评分和评价
  - 查看自己收到的评价

- [ ] **个人中心**
  - 查看个人信息和认证状态
  - 查看评分和星级
  - 完成订单数

- [ ] **客服与帮助**
  - 在线客服
  - 常见问题
  - 投诉与申诉

- [ ] **消息通知**
  - 新订单通知
  - 订单状态变更通知
  - 收入到账通知

## 五、技术架构建议

### 5.1 前端技术栈
- **框架**: React Native 0.86
- **语言**: TypeScript
- **状态管理**: Redux / Zustand / MobX
- **导航**: React Navigation
- **地图**: react-native-maps（高德/百度）
- **支付**: 微信/支付宝官方 SDK
- **即时通讯**: WebSocket / 第三方 IM（环信、融云）

### 5.2 后端技术栈（建议）
- **语言**: Node.js / Java / Python
- **框架**: Express / Spring Boot / Django
- **数据库**: MySQL / PostgreSQL + Redis
- **即时通讯**: Socket.io / WebSocket
- **地图服务**: 高德地图 API / 百度地图 API
- **支付接口**: 微信支付 / 支付宝开放平台

### 5.3 第三方服务
- **短信验证**: 阿里云短信 / 腾讯云短信
- **推送通知**: 极光推送 / Firebase Cloud Messaging
- **对象存储**: 阿里云 OSS / 腾讯云 COS（上传图片）
- **实名认证**: 阿里云实人认证

## 七、关键技术难点

1. **实时定位与轨迹追踪**
   - 司机端持续上报位置
   - 货主端实时查看
   - 需要考虑耗电和流量

2. **订单匹配算法**
   - 根据距离、车型、评分等智能匹配
   - 抢单的并发控制

3. **支付与资金安全**
   - 资金托管机制
   - 防止欺诈和刷单

4. **即时通讯的稳定性**
   - 消息送达保证
   - 离线消息处理

5. **多端数据同步**
   - 订单状态实时同步
   - 避免状态不一致

## 八、界面设计建议

### 货主端主要页面
1. 首页：快速发单入口、订单状态卡片
2. 发单页面：表单式输入货物和地址信息
3. 订单列表：待接单、进行中、已完成
4. 订单详情：司机信息、位置追踪、联系方式
5. 个人中心：个人信息、常用地址、历史订单

### 司机端主要页面
1. 首页：订单大厅、附近订单地图
2. 订单列表：待接单、进行中、已完成
3. 订单详情：货物信息、导航、联系货主
4. 收入页面：今日收入、提现
5. 个人中心：认证状态、评分、接单设置

## 九、注意事项

1. **法律合规**
   - 道路运输许可
   - 数据隐私保护（个人信息安全）
   - 资金结算合规

2. **安全性**
   - 身份认证严格
   - 防止虚假订单
   - 防止刷单作弊

3. **用户体验**
   - 操作流程简洁
   - 响应速度快
   - 错误提示友好

4. **可扩展性**
   - 后期可能增加更多车型
   - 可能扩展到其他城市
   - 预留功能扩展接口
