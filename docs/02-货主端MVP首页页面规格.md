# 货主端 MVP 首页页面规格

## 1. 页面定位

货主端首页是 MVP 的第一入口页，目标是让货主在打开 App 后能快速完成三件事：

1. 看到当前账号的认证状态和发单资格。
2. 快速进入发货流程。
3. 掌握当前订单状态，继续处理待办订单。

本页不承载完整订单发布表单，也不做复杂地图调度。第一版要把核心业务入口做扎实，别一上来就把支付、聊天、地图全炖一锅。

## 2. MVP 范围

### 2.1 本次纳入

- 货主端首页静态页面。
- 顶部用户状态区。
- 快速发单主入口。
- 订单状态概览卡片。
- 常用路线快捷入口。
- 最近订单列表。
- 客服/消息入口，点击后给出明确的暂未开放提示。

### 2.2 本次不纳入

- 登录注册真实流程。
- 实名认证提交流程。
- 完整发布订单表单。
- 地图选点、路线规划、实时定位。
- 支付、发票、聊天、推送。
- 后端接口联调。
- 司机端页面。

这些功能后续拆成独立页面和独立规格处理，别在首页里硬塞，否则首页会变成“功能垃圾桶”。

## 3. 目标用户与场景

### 3.1 目标用户

- 个人货主：临时搬运、同城配送、小批量运输。
- 企业货主：仓库发货、门店调拨、固定线路配送。

### 3.2 高频场景

1. 用户打开 App，想立刻发布一单运输需求。
2. 用户已有进行中订单，想查看司机是否接单或货物是否运输中。
3. 用户经常走同一条路线，希望快速复用地址。
4. 用户未完成实名认证，需要明确知道为什么不能发单。

## 4. 页面结构

首页从上到下分为六个区域：

1. 顶部状态栏：城市、消息、客服。
2. 用户资格卡：认证状态、发单权限、账号类型。
3. 主行动区：立即发货按钮。
4. 订单状态概览：待接单、运输中、待确认、已完成。
5. 常用路线：最多展示 2 条常用路线。
6. 最近订单：展示最近 2 到 3 条订单。

底部暂不做完整 Tab 导航；如果需要，可在实现阶段增加首页、订单、我的三个底部入口。

## 5. 视觉方向

### 5.1 设计关键词

- 工具型
- 可信赖
- 信息密度适中
- 快速决策
- 面向货运业务，不做花里胡哨的营销页

### 5.2 视觉建议

- 主色使用偏物流行业的深绿色或蓝绿色，表达稳定和运输属性。
- 辅助色使用橙色，只用于价格、提醒、关键状态。
- 背景使用浅灰，不使用大面积渐变。
- 卡片圆角控制在 8px 左右，避免做成软萌风。
- 文字层级清晰，订单信息优先可扫读。

## 6. 模块细化

### 6.1 顶部状态栏

显示内容：

- 当前城市，例如“深圳”。
- 页面标题，例如“货运发单”。
- 消息图标，显示未读数量。
- 客服图标。

交互：

- 点击城市：第一版不跳转，后续进入城市选择。
- 点击消息：第一版不跳转，后续进入消息中心。
- 点击客服：第一版不跳转，后续进入客服页。

### 6.2 用户资格卡

显示内容：

- 用户称呼，例如“下午好，张先生”。
- 账号类型：个人货主 / 企业货主。
- 认证状态：未认证 / 审核中 / 已认证 / 认证失败。
- 发单权限提示。

状态规则：

| 认证状态 | 发单按钮 | 提示文案 |
| --- | --- | --- |
| 未认证 | 禁用或弱提示 | 完成实名认证后可发布订单 |
| 审核中 | 禁用或弱提示 | 认证审核中，预计 1 个工作日内完成 |
| 已认证 | 可用 | 可发布货运订单 |
| 认证失败 | 禁用或弱提示 | 认证失败，请重新提交资料 |

MVP 第一版可以用静态 mock 数据展示“已认证”状态，避免首页一打开就全是不可用，用户体验跟一盆冷水似的。

### 6.3 快速发单主入口

显示内容：

- 主按钮：“立即发货”。
- 副文案：“填写货物和路线，快速匹配附近司机”。
- 辅助信息：预计 1 分钟完成发单。

交互：

- 点击“立即发货”：第一版展示“发布订单流程将在下一阶段接入”的轻提示。
- 如果用户未认证：提示先完成实名认证。

验收标准：

- 主按钮必须是页面最明显的行动入口。
- 不允许被订单卡片、活动 banner 抢视觉重心。

### 6.4 订单状态概览

展示四个状态入口：

- 待接单
- 运输中
- 待确认
- 已完成

每个状态显示：

- 状态名称。
- 数量。
- 简短说明。

示例数据：

| 状态 | 数量 | 说明 |
| --- | --- | --- |
| 待接单 | 2 | 等待司机报价 |
| 运输中 | 1 | 货物正在路上 |
| 待确认 | 1 | 等待确认送达 |
| 已完成 | 18 | 历史完成订单 |

交互：

- 点击状态卡：第一版不跳转，后续进入订单列表并带状态筛选。

### 6.5 常用路线

显示内容：

- 路线简称，例如“宝安仓库 → 南山门店”。
- 装货地简写。
- 卸货地简写。
- 最近使用时间。
- 快速发单按钮。

展示规则：

- 首页最多展示 2 条。
- 无常用路线时显示空状态：“暂无常用路线，完成订单后可快速复用”。

交互：

- 点击快速发单：第一版复用路线信息的能力先不做，只保留入口。

### 6.6 最近订单

订单卡片显示：

- 订单状态标签。
- 装货地到卸货地。
- 货物类型与重量。
- 车型要求。
- 订单金额。
- 更新时间。
- 快捷操作按钮。

不同状态的主按钮：

| 状态 | 主按钮 |
| --- | --- |
| 待接单 | 查看报价 |
| 待装货 | 联系司机 |
| 运输中 | 查看位置 |
| 待确认 | 确认送达 |
| 已完成 | 再来一单 |

MVP 第一版只做静态列表和按钮展示，不实现真实订单流转。

## 8. 空状态与异常状态

### 8.1 无最近订单

文案：

- 标题：“还没有订单”
- 说明：“发布第一单后，可在这里查看运输进度”
- 操作：“立即发货”

### 8.2 无常用路线

文案：

- 标题：“暂无常用路线”
- 说明：“完成订单后，常用线路会显示在这里”

### 8.3 未认证

文案：

- 标题：“完成实名认证后可发货”
- 说明：“平台需要核验身份，保障交易安全”
- 操作：“去认证”

## 9. 可访问性与移动端约束

- 主要按钮高度不低于 44px。
- 可点击区域不低于 44px。
- 订单卡片信息必须支持小屏换行，不允许文字挤出容器。
- 金额、状态、地址要有明确视觉层级。
- 深色模式第一版暂不做，但颜色不能写死到难以后续扩展。

## 10. 埋点需求

第一版预留以下事件命名，暂不接入真实埋点 SDK：

| 事件名 | 触发时机 |
| --- | --- |
| `shipper_home_viewed` | 进入货主首页 |
| `shipper_home_create_order_clicked` | 点击立即发货 |
| `shipper_home_status_clicked` | 点击订单状态卡 |
| `shipper_home_route_reuse_clicked` | 点击常用路线快速发单 |
| `shipper_home_recent_order_clicked` | 点击最近订单 |

## 11. 验收标准

页面完成后必须满足：

1. App 启动后展示货主端首页，不再展示 React Native 默认模板页。
2. 首页包含顶部状态栏、用户资格卡、立即发货入口、订单状态概览、常用路线、最近订单。
3. 页面使用本地 mock 数据，不依赖后端接口。
4. 页面在常见 Android 手机宽度下不出现文字重叠或按钮挤压。
5. `npm run lint` 通过。
6. `npx tsc --noEmit` 通过。
7. Jest 配置问题修复后，首页组件应补充渲染测试。
