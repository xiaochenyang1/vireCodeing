# 货车接单平台 - 需求分析文档

## 项目概述

开发一个双端的货车接单平台Android应用，连接货主和车主，实现货运订单的发布、接单、配送全流程管理。

---

## 一、用户角色定义

### 1.1 货主端（发货方）
- **角色定位**：需要运输货物的个人或企业用户
- **核心需求**：发布货运需求、选择车主、跟踪物流

### 1.2 车主端（承运方）
- **角色定位**：拥有货车的司机或运输企业
- **核心需求**：接收订单、运输货物、获得收益

## 五、业务流程

### 5.1 货主发单流程
```
货主注册/登录 
  → 完成实名认证 
  → 填写货物信息 
  → 填写起始地和目的地 
  → 设置运费 
  → 发布订单 
  → 等待车主接单 
  → 确认车主 
  → 支付运费（预付/到付） 
  → 跟踪物流 
  → 确认收货 
  → 评价车主
```

### 5.2 车主接单流程
```
车主注册/登录 
  → 完成实名认证 
  → 完成车辆认证 
  → 浏览订单大厅 
  → 查看订单详情 
  → 抢单/接受邀请 
  → 联系货主 
  → 前往装货地点 
  → 装货并拍照 
  → 开始运输 
  → 实时上报位置 
  → 到达目的地 
  → 卸货并拍照 
  → 确认完成 
  → 收款 
  → 评价货主
```

### 5.3 订单状态流转
```
待接单 → 待确认 → 待装货 → 运输中 → 待签收 → 待评价 → 已完成
   ↓         ↓         ↓         ↓
 已取消   已取消    已取消    异常订单
```
